This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Development Commands

### Docker (recommended)
```bash
docker compose up --build          # start everything
docker compose logs -f             # tail logs
docker compose down                # stop
curl http://localhost:8000/api/gpu-info   # verify GPU/acceleration status
```

### Local development (backend + frontend separately, with hot-reload)
```bash
# Backend (from /backend)
pip install -r requirements.txt
uvicorn main:app --reload --host 127.0.0.1 --port 8000

# Frontend (from /frontend)
npm install
npm run dev        # Vite dev server on port 5173
npm run build      # build into frontend/dist (served by FastAPI in production)
```

The Vite dev server proxies nothing — CORS is already configured in `main.py` for `localhost:5173`. In dev mode the frontend talks directly to `localhost:8000`.

### API docs
`http://localhost:8000/api/docs` (Swagger UI auto-generated by FastAPI)

## Architecture

### Request/session lifecycle
1. **Upload** — `POST /api/upload` creates a UUID session directory, streams the E57 file in, calls `e57_handler.read_e57()` to parse all scans into a single `(N,3)` xyz array and `(N,3)` rgb float32 array, and saves both as `original.npz` + `current.npz`. A downsampled binary preview (`preview_orig.bin`) is also generated.
2. **Process** — `POST /api/process/auto/{id}` or `/manual/{id}` dispatches to `noise_removal.py` in a background thread (via FastAPI `BackgroundTasks` + `run_in_executor`). Results are written back to `current.npz` and a fresh `preview_curr.bin`. Every operation is appended to `audit.db` (SQLite, insert-only).
3. **Preview** — `GET /api/process/preview/{id}/original|current` serves the raw binary: interleaved `float32[x,y,z,r,g,b]` × N points (max 2 M after voxel downsampling). `Viewer3D.tsx` reads this with a `Float32Array` and builds Three.js `BufferGeometry`.
4. **Export** — `GET /api/export/{id}` calls `e57_handler.write_e57()` to reconstruct a valid E57 from `current.npz`.
5. **Audit report** — `GET /api/audit/{id}/report` reads `audit.db` and generates a PDF via `pdf_generator.py` (ReportLab).

### Session storage layout (`WORK_DIR/forensiccloud_sessions/{uuid}/`)
| File | Purpose |
|------|---------|
| `original.e57` | Uploaded file (untouched) |
| `original.npz` / `current.npz` | Numpy arrays `xyz` (float64) + `rgb` (float32) |
| `preview_orig.bin` / `preview_curr.bin` | Downsampled float32 binary for the browser |
| `audit.db` | SQLite audit log (immutable inserts) |
| `meta.json` | Status, filenames, point counts, bbox |

### GPU acceleration (gpu_utils.py)
`gpu_utils.py` is probed once at import time. It exports:
- `GPU_AVAILABLE` (bool), `xp` (cupy or numpy), `to_gpu()`, `to_cpu()`, `free_gpu_bytes()`
- `SKLEARN_AVAILABLE` (bool), `CPU_COUNT` (int)

All algorithms follow a three-tier dispatch — GPU (CuPy) → CPU parallel (sklearn / `ProcessPoolExecutor`) → serial (scipy `cKDTree`). New algorithms must follow this same pattern and use `xp` (not `np` directly) for GPU-compatible math.

### Algorithms (noise_removal.py)
Every public function returns a `RemovalResult` dataclass with `kept_xyz`, `kept_rgb`, `removed_count`, `removed_bbox`, and `removed_sample`. The router calls `_commit_result()` to persist results and build the next preview.

Auto-clean pipeline order: SOR → Color Filter → Plane RANSAC → ROR.

`plane_ransac_removal` uses a module-level `_ransac_worker` function (required for `ProcessPoolExecutor` pickling on Windows spawn context).

### Frontend (React + Three.js)
- `App.tsx` owns all session state and polls `/api/process/status/{id}` every 1500 ms. Status transitions (`loading→ready`, `processing→ready`) drive preview reloads via incrementing `origKey`/`currKey`.
- `Viewer3D.tsx` renders a split-screen: left half = original cloud, right half = current (cleaned). Both share one `OrbitControls` / camera. Selection mode captures two raycasted clicks on the left half to produce a `BoundingBox`.
- `frontend/src/api/client.ts` is the single API boundary — all fetch calls go through here.
- Tailwind classes use custom tokens (`bg-bg`, `bg-surface`, `text-accent`, `text-muted`, `text-success`, `text-dim`, `border-border`) defined in `tailwind.config.js`.

### Key constants (config.py)
- `MAX_PREVIEW_POINTS = 2_000_000` — browser point budget
- `PREVIEW_VOXEL_SIZE = 0.02` — voxel size (metres) for downsampling
- `FORENSICCLOUD_WORK_DIR` env var overrides the session scratch directory
